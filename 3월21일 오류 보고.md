# 🎧 리스닝 마스터 앱 - 오류 보고서
> **날짜**: 2026년 3월 21일  
> **분석 방법**: LLM 코드 리뷰 (전체 소스 코드 및 데이터 파일 분석)

## 🔴 A. 버그 (코드 오류)

### A1. 어휘 퀴즈 결과의 "맞힘" 오타
- **위치**: `vocab.js` 223~224행, `results.js` 63행
- **문제**: `맞힘` → 올바른 표현은 `맞힌 수` 또는 `정답`
- **영향**: 사용자에게 어색한 한국어 표현 노출

### A2. 듣기 모의고사 재생 횟수 1회 제한 — 너무 엄격
- **위치**: `test.js` 117행
- **문제**: `canPlay = plays < 1` → 딱 1번만 재생 가능. 실제 시험에서도 보통 2회 재생을 허용
- **영향**: 한 번 실수로 클릭하면 다시 들을 수 없음 → 학습 목적에 부적합

### A3. 5개 이상 선택지 있는 문제에서 키보드 숏컷 누락
- **위치**: `test.js` 304행
- **문제**: 키보드 선택은 1~4만 지원
- **영향**: 현재 데이터는 4지선다이므로 당장은 괜찮으나, 확장성 부족

### A4. 결과 페이지 "다시 듣기" 오디오 에러 무시
- **위치**: `results.js` 173~183행
- **문제**: `audio.play().catch(() => {})` — 오류가 발생해도 사용자에게 아무 피드백 없음
- **영향**: 사용자가 소리가 안 나는 이유를 모름

### A5. 받아쓰기(Dictation) 데이터가 전체 10회차 전부 비어있음
- **위치**: 모든 `round1.json` ~ `round10.json`의 `dictation` 필드
- **문제**: `step1: [], step2: [], step3: []` — 모든 회차의 받아쓰기 데이터가 빈 배열
- **영향**: 받아쓰기 기능이 **완전히 사용 불가**

### A6. 받아쓰기 오디오 URL에 확장자 누락
- **위치**: `dictation.js` 172행
- **문제**: 프로덕션 빌드 시 Vite 미들웨어가 없으면 오디오 작동 안 함
- **영향**: 개발 환경에서만 작동, 정적 배포 시 오디오 불가

### A7. 키보드 핸들러 정리 불완전
- **위치**: `main.js` 49~53행
- **문제**: `_homeKeyHandler`만 정리하고, vocab/test/dictation의 핸들러는 미정리
- **영향**: 잠재적 메모리 누수 및 키 중복 동작

### A8. 어휘 데이터에 잘못된 데이터 항목 존재
- **위치**: `round5.json` 123~129행
- **문제**: `"in  ~"`, `"have to  ~해야"` → 올바른 영어 단어가 아님
- **영향**: TTS에서 이상한 발음, 퀴즈에서 혼란

### A9. Round 5 Q8에 `questionImage` 필드 누락
- **위치**: `round5.json` 299행
- **문제**: `imageQuestion: true`이지만 `questionImage` 필드 없음
- **영향**: 이미지형 문제인데 그림이 안 보임

### A10. TTS 스크립트 줄바꿈 파싱 불일치
- **위치**: `test.js` 37행 vs `results.js` 123행
- **문제**: test.js에서는 `\\\\n`으로 split, results.js에서는 `/\\n/g`으로 replace
- **영향**: 결과 페이지 스크립트 표시가 한 줄로 나올 수 있음

## 🟡 B. 사용성 / UX 불편사항

### B1. 어휘 학습 "완료" 버튼이 마지막 카드에서만 보임
- 모든 카드를 순서대로 다 넘겨야만 완료 가능

### B2. 모의고사 중 문제 번호 점프 불가
- 특정 문제로 바로 이동 불가, 이전/다음만 가능

### B3. 모의고사 "결과 보기" 버튼이 모든 문제를 풀어야 나타남
- 일부만 풀어도 결과를 보고 싶을 수 있음

### B4. 페이지 전환 시 TTS가 겹칠 수 있음
- 빠르게 카드를 넘기면 발음이 겹침

### B5. 받아쓰기 정답 확인 후 자동 이동 없음
- 다음 문제로 수동 이동 필요

### B6. 받아쓰기 완료 기준이 너무 관대함
- step1 하나만 해도 완료 처리됨

### B7. 점수 표시 방식
- 백분율보다 "10/15" 같은 직관적 표현이 초등학생에게 적합할 수 있음

### B8. 어휘 퀴즈 선택지 4개 미만 가능성
- 데이터가 적으면 선택지가 부족해질 수 있음

### B9. 데이터 초기화 기능이 UI에 없음
- `store.reset()` 함수는 있지만 호출 버튼 없음
